# Концепция программного проекта

## Название проекта

**«ИС учёта и обработки заявок IT-поддержки государственного учреждения»**

---

# 1. Описание проблемы

| Пункт | Описание |
|---|---|
| Проблема | В государственных учреждениях обработка заявок в IT-отдел часто ведётся через телефон, мессенджеры или устные обращения. Из-за этого заявки теряются, отсутствует контроль сроков выполнения и сложно отслеживать загрузку IT-специалистов. |
| Воздействует на | Сотрудников учреждения, IT-специалистов, руководство подразделения. |
| Результатом чего является | Увеличение времени решения технических проблем, потеря заявок, отсутствие прозрачности работы IT-подразделения, снижение эффективности сотрудников учреждения. |
| Выигрыш от ПП | Централизованный учёт заявок, контроль статусов выполнения, распределение нагрузки между специалистами, повышение скорости обработки обращений, хранение истории заявок и улучшение качества IT-поддержки. |

---

# 2. Назначение программного средства

## Какие системы смогут применять ПС для решения своих задач?

Программное средство может использоваться:

- IT-подразделениями государственных учреждений;
- отделами технической поддержки организаций;
- административными подразделениями;
- небольшими и средними организациями с внутренней IT-поддержкой.

---

## От каких систем/пользователей поступают входные данные ПС?

### Пользователи системы:

1. Сотрудник учреждения
   - создаёт заявки;
   - отслеживает статус обращения;
   - получает уведомления о выполнении.

2. IT-специалист
   - принимает заявки в работу;
   - изменяет статусы;
   - оставляет комментарии;
   - закрывает обращения.

3. Администратор системы
   - управляет пользователями;
   - назначает роли;
   - контролирует работу системы;
   - просматривает статистику.

---

## Кто/что использует результаты?

- сотрудники учреждения;
- IT-отдел;
- руководство подразделения;
- администрация организации.

---

## Какие сторонние системы/сервисы участвуют в процессе обработки данных?

- база данных PostgreSQL;
- почтовый сервис уведомлений;
- операционная система сервера;
- веб-браузер пользователя.

# 5. Описание программного продукта по шаблону

Для сотрудников и IT-подразделений государственных учреждений, которым требуется удобный инструмент учёта и обработки технических обращений, данный программный продукт «ИС учёта и обработки заявок IT-поддержки государственного учреждения» является системой автоматизации обработки IT-заявок, которая позволяет централизованно регистрировать, отслеживать и контролировать выполнение обращений сотрудников.

В отличие от устных обращений, телефонных звонков и переписки в мессенджерах, наш продукт обеспечивает прозрачный контроль заявок, хранение истории обращений и повышение эффективности работы IT-подразделения.
